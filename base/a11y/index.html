Accessibility

Rocketbelt strives to be accessible out of the box.

Accessibility Checklist

Principle Rule Guidance
Text Alternatives All images have appropriate, equivalent alternative text. Ensure that alt text is accurate, succinct, not redundant, and doesn't use phrases like "Image of" or "Graphic of".
Text Alternatives Images that do not convey content, are decorative, or contain content that is already conveyed in text are given null alt text (alt="") or implemented as CSS backgrounds. All linked images have descriptive alternative text. Ensure alt text doesn't just duplicate existing text.
Text Alternatives Form buttons have a descriptive value. A button with no text value should have a title tag that indicates what it does.
Text Alternatives Form inputs have associated text labels. Ensure that every form label is linked to its input with a for attribute.
Text Alternatives Embedded multimedia is identified via accessible text. Add a aria-label or aria-labelledby attributes that indicate what the content is.
Text Alternatives Frames are appropriately titled.
Adaptable Semantic markup is used appropriately to designate headings, lists, areas of the page (using HTML5 semantic tags such as section, aside, etc.) and emphasized or special text. Semantic markup eliminates the need to add aria-role attributes.
Adaptable Tables are used for tabular data. Where necessary, data cells are associated with their headers. Data table captions and summaries are used where appropriate.
Adaptable Text labels are associated with form input elements. Related form elements are grouped with fieldset/legend. Ensure that every form label is linked to its input with a for attribute. Group related elements into a fieldset when appropriate.
Adaptable The reading and navigation order (determined by code order) is logical and intuitive.
Adaptable Instructions do not rely upon shape, size, or visual location. Content may flow differently at various viewport widths—avoid phrases like "on the right," "see the note beneath," etc.
Distinguishable Color is not used as the sole method of conveying content or distinguishing visual elements. Use color in addition to another visual indicator such as text or an icon.
Distinguishable The page is readable and functional when the text size is increased. Change the browser's zoom level with Ctrl-/Cmd-+. Does the page break? Are its aesthetics maintained?
Distinguishable If the same visual presentation can be made using text alone, an image is not used to present that text. Avoid embedding meaningful text in images at all costs. If it cannot be avoided, use alt text or a visually-hidden element linked with an aria-describedby attribute on the image.
Distinguishable Text is used within an image only for decoration (the image does not convey content) or when the information cannot be presented with text alone. Avoid embedding meaningful text in images at all costs. If it cannot be avoided, use alt text or a visually-hidden element linked with an aria-describedby attribute on the image.
Navigable All page functionality is available using the keyboard. Use the TAB key to navigate from element to element. Ensure all elements that can be interacted with via mouse can receive keyboard focus. Ensure all anchor tags have attributes.
Navigable A link is provided to skip navigation and other page elements that are repeated across web pages. Implement Rocketbelt's Skip Link pattern as the first focusable element on the page.
Navigable The purpose of each link can be determined from the link text alone, or from the link text and the content that surrounds it. Avoid unclear link text like "click here." Add a title attribute to any text that may be unclear.
Navigable Links (or form image buttons) with the same text that go to different locations are readily distinguishable. Use descriptive, clear language in link text. Add a title attribute to any text that may be unclear.
Navigable It is visually apparent which page element has the current keyboard focus (i.e., as you tab through the page, you can see where you are). Never remove CSS outlines. Favor minimal outlining such as Rocketbelt's default (outline: thin dotted;).
Navigable The purpose of each link (or form image button or image map hotspot) can be determined from the link text alone. Use descriptive, clear language in link text.
Navigable There are no links (or form image buttons) with the same text that go to different locations. Use descriptive, clear language in link text.
Predictable When a page element receives focus, it does not result in a substantial change to the page, the spawning of a pop-up window, an additional change of keyboard focus, or any other change that could confuse or disorient the user. Allow the user to trigger these states instead. Do not use Javascript to steal keyboard focus.
Predictable When a user interacts with the page, it does not result in a substantial change to the page, the spawning of a pop-up window, an additional change of keyboard focus, or any other change that could confuse or disorient the user unless the user is informed of the change ahead of time. Ensure that any link or button that triggers a dialog indicates this behavior with its text or via a title tag. Do not use Javascript to steal keyboard focus.
Predictable Substantial changes to the page, the spawning of pop-up windows, uncontrolled changes of keyboard focus, or any other change that could confuse or disorient the user must be initiated by the user. Ensure that any link or button that triggers a dialog indicates this behavior with its text or via a title tag. Do not use Javascript to steal keyboard focus.
Compatible Markup is used in a way that facilitates accessibility. Significant HTML/XHTML validation/parsing errors are avoided. Use the W3C's validation service or validation bookmarklet.

Accessibility User Stories

  • As a keyboard-only user, I want to be able to reach navigation links with a keyboard so that I can access different areas of the site.
  • As a keyboard-only user, I want the ability to reach all links (text or image), form controls and page functions, so that I can perform an action or navigate to the place I choose.
  • As a keyboard-only user, I want the ability to use the enter key to open the selected link so that every link on a page is accessible using a keyboard as it would be with a mouse click.
  • As a keyboard-only user, I want to know where I am on the screen at all times so that I know what I can do and how to do it.
  • As a screen reader user, I want to hear the text equivalent for each image conveying information so that I don't miss any information on the page.
  • As a screen reader user, I want to understand know what each form label is for each form field so that I can effectively enter the correct information in the form.

Skip Links

Main content or items in a page's footer may require dozens or hundreds Tab presses to interact with if a user is navigating via keyboard. Skip links make this interaction much less tedious by using an on-page href. The link is visually hidden unless it is focused via keyboard. 

                              <ul class="skip list-reset">
                                <li><a class="button-sm button-secondary" href="#content">Skip to Main Content</a></li>
                                <li><a class="button-sm button-secondary" href="#footer">Skip to Footer</a></li>
                              </ul>
