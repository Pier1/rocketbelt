## Make Iconography Accessible
### Meaningful Icon
Use this pattern when an icon's meaning is clear and the icon is used as an
image (not for an action).
- Note the `role=img` attribute
- Icon description is available to assistive technologies via the
`aria-describedby` attritube in the icon sprite sheet

```html
<svg class="icon" role="img">
  <use xlink:href="/icons/rocketbelt.icons.svg#rb-icon-chat-single"></use>
</svg>
```

### Icon with Text
Use this pattern when an icon's meaning can be inferred by the text around it and
the icon is purely decorative.
- Note the use of the `aria-hidden` attributeâ€”the text should convey all necessary information

```html
<span class="text-with-icon">Store Locator
  <svg class="icon icon-left" aria-hidden>
    <use xlink:href="/icons/rocketbelt.icons.svg#rb-icon-pin"></use>
  </svg>
</span>
```

### Linked Icon
Use the Linked Icon pattern when an icon has an implicit action that isn't conveyed by it's `aria-describedby`
attribute (which is a literal description of the icon, similar in nature to `alt` text). An `aria-label` is necessary to help describe the intent (not content) of the icon

```html
<a href="/">
<svg class="icon" aria-label="Go to the home page" role="img">
  <use xlink:href="/icons/rocketbelt.icons.svg#rb-icon-home"></use>
</svg></a>
```

### Icon with Overlay

```html
<a class="text-with-icon" href="#"><span class="indicator">3</span><span class="visually-hidden">items in your cart</span>
<svg class="icon" aria-hidden>
  <use xlink:href="/icons/rocketbelt.icons.svg#rb-icon-cart"></use>
</svg></a>
```


## Standard Icons

- An icon's `fill` property is inherited from `color`
- Always use the default color for logo icons

import rbIcons from '../../../rocketbelt/components/icons/rocketbelt.icons.svg';
import IconPresenter from '../../../components/icon-presenter';

<IconPresenter svgSprite={rbIcons} />
