# Accordions, Expanders & Tabs

Use these components to focus a user's attention on a page's more primary
elements. The **progressive disclosure** provided by these components simplifies a
page's visual weight while providing an information scent by which users can
find content of secondary importance.

Avoid using these components if the user may need most or all of a page's
content to make a decision.[^1] Hidden content comes at the cost of awareness
and cognitive load.

If the contents of an Accordion, Expander, or Tab group contain <Link to="/components/forms">Form components</Link>,
be certain that the form's state is explicitly clearâ€”specifically the state it
will be in if the user collapses the Accordion or switches to a different
Tab.[^2]

## Expanders
Choose an Expander if you only need to collapse one section of information. Only
place Expanders adjacent to one another if the information in one expander isn't
logically related to information in the other.

```html
<aside class="expandable">
  <span class="expander">
    Section 1
  </span>
  <section>
    <p>
      Lorem ipsum dolor sit amet, consectetur adipiscing elit. Quisque commodo
      purus quis mi cursus hendrerit eu eu metus. Aliquam aliquam arcu eget
      aliquet scelerisque. Pellentesque sodales turpis vitae venenatis vehicula.
      Ut id porta velit. Ut eu dignissim dui, quis gravida est. Cras quis
      venenatis mauris, a bibendum enim. Sed at augue libero. Nullam tortor
      metus, tincidunt ut urna id, posuere placerat orci. Ut quis risus dictum
      risus facilisis imperdiet quis sed eros.
    </p>
  </section>
</aside>
```

## Accordions

Accordions should be used to collapse multiple sections of information that are
logically related to one another.

```html
<aside class="accordion">
  <span class="expander">
    Section 1
  </span>
  <section>
    <p>
      Lorem ipsum dolor sit amet, consectetur adipiscing elit. Quisque commodo
      purus quis mi cursus hendrerit eu eu metus. Aliquam aliquam arcu eget
      aliquet scelerisque. Pellentesque sodales turpis vitae venenatis vehicula.
      Ut id porta velit. Ut eu dignissim dui, quis gravida est. Cras quis
      venenatis mauris, a bibendum enim. Sed at augue libero. Nullam tortor
      metus, tincidunt ut urna id, posuere placerat orci. Ut quis risus dictum
      risus facilisis imperdiet quis sed eros.
    </p>
  </section>
  <span class="expander">
    Section 2
  </span>
  <section>
    <p>
      Nullam tortor metus, tincidunt ut urna id, posuere placerat orci. Ut quis
      risus dictum risus facilisis imperdiet quis sed eros. Lorem ipsum dolor
      sit amet, consectetur adipiscing elit. Quisque commodo purus quis mi
      cursus hendrerit eu eu metus. Aliquam aliquam arcu eget aliquet
      scelerisque. Pellentesque sodales turpis vitae venenatis vehicula. Ut id
      porta velit. Ut eu dignissim dui, quis gravida est. Cras quis venenatis
      mauris, a bibendum enim. Sed at augue libero.
    </p>
  </section>
</aside>
```

## Tabs
Choose a Tab component when the information that each tab contains is logically
related but that the user is unlikely to need at the same time.

    </li>

    </li>
    <li>
      <button class="tabs_tab button button-minimal">Section 3</button>
    </li>
     <li>
